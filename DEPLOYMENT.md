This repository contains a FastAPI backend and a TanStack Start (Vite + React) frontend.

This document summarizes steps to deploy the project locally, with Docker, and guidance for cloud.

1) Local development (recommended for initial testing)

Prereqs:
- Python 3.11+
- Node.js 18+ (or Bun) and npm
- Git

Backend (local):
- cd backend
- python -m venv venv
- On Windows: .\\venv\\Scripts\\Activate.ps1 (PowerShell) or venv\\Scripts\\activate (cmd)
- On macOS/Linux: source venv/bin/activate
- pip install -r requirements.txt
- Copy `backend/.env` to `backend/.env.local` and set secrets (GROQ_API_TOKEN, TAVILY_API_KEY, etc.) if needed.
- Run the dev server:

  python -m fastapi dev app/api/main.py

- Backend available at http://localhost:8000 (API docs: /docs)

Frontend (local):
- cd frontend
- npm install
- Ensure `PORT` env is set to 8080 (default used by `startup` scripts)
- npm run dev
- UI available at http://localhost:8080

Use the provided `startup.ps1` or `startup.bat` to launch both services on Windows.

2) Docker (template / recommended)

This repo currently lacks Dockerfiles / docker-compose. Recommended minimal setup:

- Add `backend/Dockerfile` using Python 3.11-slim, copy requirements and source, install, expose 8000, run `uvicorn app.api.main:app --host 0.0.0.0 --port 8000 --workers 1`.
- Add `frontend/Dockerfile` that builds the Vite app and serves the `dist` with an nginx static server on port 80.
- Add `docker-compose.yml` at repo root with services: `backend`, `frontend`, and optionally an S3-compatible service (MinIO) for local integration and a Chroma service if needed.

Example docker-compose tips:
- Map ports 8000 -> backend, 8080 -> frontend (or serve frontend on 80)
- Forward environment variables via `.env` file
- Use `depends_on` for ordering; configure healthchecks

3) Cloud deployment guidance

- Backend: deploy to any container-based platform (AWS ECS, EKS, GCP Cloud Run, Azure App Service, Render, Fly). Build and push backend image; set environment variables (GROQ_API_TOKEN, FILE_STORAGE_BACKEND, R2 credentials if using S3/R2)
- Frontend: deploy static build (vite build) to static hosting (Netlify, Vercel, Cloudflare Pages) or containerize and serve via nginx.
- Storage: configure S3 or Cloudflare R2 and set `FILE_STORAGE_BACKEND=s3` with `S3_BUCKET`, `R2_*` credentials. Ensure network ACLs and permissions allow access.

4) Environment variables

Key env vars found in `backend/.env`:
- `TAVILY_API_KEY`
- `GROQ_API_TOKEN` (or GROQ_API_KEY)
- `FILE_STORAGE_BACKEND` (local or s3)
- `R2_ENDPOINT_URL`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY` (for R2/S3-compatible storage)
- `S3_BUCKET`, `S3_BASE_PREFIX`
- Optional: `TASK_TTL_SECONDS`, `TASK_TIMEOUT_SECONDS`, `CORS_ALLOW_ORIGINS`

5) Security and production notes
- Don't commit `.env` with secrets. Use secret stores (Vault, GitHub Secrets, platform env settings).
- Use a production ASGI server: `uvicorn` or `gunicorn` with `uvicorn.workers.UvicornWorker` behind a reverse proxy.
- Configure CORS for the frontend domain.
- Use HTTPS in production and secure S3 credentials.
